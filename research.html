Usability Testing


Usability is one of many important quality attributes that apply to your phone application. Usability can be split into 5 key components: Learnability, Efficiency, Memorability, Errors and Satisfaction. (Nielsen , 1999)

  • Learnability is how easy it is for a user to accomplish a basic task the first time they are presented with the design (Nielson, 2012).
  • Efficiency is once a user has learnt the design of the application how quickly can they can perform the tasks (Nielson, 2012).
  • Memorability is when a user returns to the application after a period of not using it, how quickly that can relearn it (Nielson, 2012).
  • Errors is the amount of errors the user makes, how many errors they make, the severity of the errors and how they recover from the errors are all factors to be considered (Nielson, 2012).
  • Satisfaction refers to how pleasant the user finds the application (Nielson, 2012).


Why Usability Test?



Usability testing allows the design and development team to identify problems within the application before it is released. The earlier issues are identified the less expensive they are in cost as well as man hours (usability.gov, 2015).

Usability testing can allow you to learn:

  • If participants can successfully complete tasks
  • How long specific tasks take
  • How satisfied participants are with your application
  • What changes need to be made to improve users satisfaction


Methods of Usability Testing



There are multiple techniques that can be used to run a Usability Test. Once the test has been planned and the test participants have been recruited a moderating technique has to be chosen that is suitable for the test. (Bergstrom, 2013)

Some techniques that can be used are:

Concurrent Think Aloud

The participant speaks aloud their every step and thought.

Retrospective Think Aloud

When the session is complete the participant is asked to retrace all the steps they completed during the test

Concurrent Probing

The participants work on tasks and when they say or do something interesting the researcher takes notes.

Retrospective Probing

After the session is complete the participant is asked questions about the test conducted. This is usually used in conjunction with other methods.

Pilot Test

Before any testing is done a pilot test must be conducted. This test allows you to check the following:

  • The equipment
  • The researcher can practice note taking
  • A final check that tasks are clear
  • Allow you to make any last minute changes
